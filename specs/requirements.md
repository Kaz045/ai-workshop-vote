# 要件定義

## 現行スコープ

このリポジトリで現在確定しているスコープは、次の 4 つです。

1. With ポータルの公開トップページを安定して公開すること
2. 第1回ワークショップの会員向けアーカイブを公開すること
3. 旧投票プロジェクトを `archive/` に退避し、履歴として保持すること
4. 新しい運用判断を `specs/` とルート文書で追えること

## 機能要件

### RQ-001 公開トップページ

- `site/index.html` は With ポータルの入口として機能する
- 第1回アーカイブへの導線を持つ
- 今後の拡張予定を簡潔に示す
- モバイルでも可読性を保つ

### RQ-002 会員向けアーカイブページ

- `site/workshops/session-1.html` はページ単位のパスワード保護を持つ
- パスワード一致後にコンテンツを表示する
- 第1回ワークショップの振り返り、改善点、関連資料の扱いを表示する
- 同一ブラウザセッション中は再入力なしで閲覧できる

### RQ-003 履歴資産の保持

- 旧公開ページは `archive/v1-vote-result/` に残す
- 旧投票プロトタイプは `archive/html_experiment/` に残す
- 旧仕様書は `archive/specs/` に残す
- 旧 Firebase 設定は `archive/firebase/` に残す

### RQ-004 ドキュメントの再整理

- ルート文書と `specs/` で現在の正史を辿れること
- `archive/` と `docs/` は履歴・参考レーンとして明示されること
- 重い原本データは `records/` に置き、Git 管理から分離すること

## 非機能要件

### NF-001 実装方式

- フレームワークを使わず、静的 HTML + CSS + vanilla JS で構成する
- ビルド不要で GitHub Pages へデプロイできること

### NF-002 運用性

- 第2回以降も同じ構造でページを追加しやすいこと
- 共通 CSS と共通パーツを再利用できること

### NF-003 セキュリティ

- ページ保護はライトな閲覧制御として扱う
- 機密情報を置く用途には使わない
- パスワード本体はソースに平文で置かない

### NF-004 AI 可読性

- 新規エージェントが 5 分以内に現在地を把握できる粒度で要点がまとまっていること

## 既知のギャップ

- 議事録 PDF の外部リンク先はまだ未確定
- ページ単位の保護は正式認証ではない
- 検索導線やタグ導線は今後のコンテンツ量を見て判断する
