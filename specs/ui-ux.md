# UI / UX

## 現在の公開フロー

### 公開トップページ

1. 閲覧者が GitHub Pages の URL を開く
2. With ポータルの目的を短時間で理解する
3. 第1回アーカイブへの導線を見つける
4. 今後の追加予定を把握する

### 会員向けアーカイブページ

1. 会員がページを開く
2. パスワード入力欄を確認する
3. 認証後に開催後レポートの要点を見る
4. 投票結果や関連資料の扱いを確認する

## 主要画面

### 画面名: ポータルトップ

- 役割: ポータルの入口
- 主要要素:
  - ヒーロー
  - アーカイブカード
  - これまでの流れ
  - 今後の予定
- 主要アクション: 第1回アーカイブへ進む

### 画面名: 第1回アーカイブ

- 役割: 会員向け振り返り
- 主要要素:
  - ページ保護
  - 振り返りポイント
  - 改善の方向
  - 投票結果の再掲
  - 関連資料の扱い
- 主要アクション: 内容を読む

## UX 方針

- 最初に「何のページか」が分かることを優先する
- 説明を盛り込みすぎず、読み進めやすい粒度に整える
- ページ保護は邪魔になりすぎず、でも会員向けページだと分かる見せ方にする
- スマホでも読みやすいことを最優先にする
