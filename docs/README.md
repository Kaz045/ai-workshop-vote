# docs ガイド

`docs/` は、現在のワークスペース構成を導入する前からある資料を保管する参考レーンです。
ここにある文書は履歴確認や背景把握には使えますが、実装・運用の正史ではありません。

現在の正史はルート文書と `../specs/` にあります。

## このフォルダの位置づけ

- `docs/` は履歴・参考・輸入資料の保管場所
- 新しい承認済み仕様は `../specs/`
- 新しい生メモは `../ideas/`
- 新しい企画候補は `../proposals/`
- 日次の作業ログは `../sessions/`

## サブフォルダ

### `plans/`

旧企画書、構成案、進行メモの保管場所です。
今後も参照はできますが、採用した内容は `proposals/` や `specs/` へ昇格させます。

### `chatgpt_exports/`

ChatGPT の会話エクスポート置き場です。
発想の源泉としては有用ですが、直接の実装指示としては扱いません。

### `operations/`

運用、障害、セキュリティ対応のメモです。
実運用の背景確認に使います。
