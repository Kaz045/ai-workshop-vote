# docs ガイド

`docs/` は文章ドキュメント専用のフォルダです。
役割ごとに以下の3つへ分けています。

## フォルダ構成

### `plans/`

企画書、構成案、進行中メモを置く場所です。

ここに入れるもの:

- ワークショップ企画書
- 次回開催メモ
- 告知文の下書き
- 進行構成案

今ある主なファイル:

- `plans/ワークショップ企画書_v1.0.md`
- `plans/ワークショップ企画書_v1.5.md`
- `plans/次回ワークショップ企画メモ_2026-03-15.md`

### `chatgpt_exports/`

ChatGPT の会話を書き出した資料を置く場所です。
元の発想や検討過程をあとから見返したいときに使います。

ここに入れるもの:

- ChatGPT の会話エクスポート
- 元の壁打ちログ
- 発想のたたき台として残したい対話記録

### `operations/`

運用、障害対応、セキュリティ対応のメモを置く場所です。

ここに入れるもの:

- インシデント対応メモ
- デプロイ運用メモ
- 障害対応手順

今ある主なファイル:

- `operations/firebase_api_key_incident_response.md`

## 迷ったときの置き場

- 企画を進めるための文章なら `plans/`
- AIとの会話ログなら `chatgpt_exports/`
- 公開運用や障害対応のメモなら `operations/`

## 追加ルール

- 新しい企画メモは基本的に `plans/` に置く
- ChatGPT の書き出しは `chatgpt_exports/` にまとめる
- 公開ページの運用メモやセキュリティメモは `operations/` に置く
