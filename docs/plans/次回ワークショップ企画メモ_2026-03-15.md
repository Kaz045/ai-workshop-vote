# 次回ワークショップ企画メモ

作成日: 2026-03-15  
更新日: 2026-03-15

## 位置づけ

リアルタイム投票の受付終了後に、
最初に開催する実ワークショップ企画を具体化するためのメモ。

投票結果を見る限り、今回は単発1回よりも
**入門編 + 中級編の2回シリーズ**として設計する方が自然。

## 投票結果サマリー

- AIでツールを作る（Vibe Coding）: 11票
- AI開発の世界を覗く（VS Code / Cursor / GitHub）: 7票
- ChatGPTを仕事の相棒にする: 4票
- 企業でのAI導入とCopilot: 1票

### この結果から見えること

- 参加者の関心は「AIで何かを作ること」と「AI開発の世界観を知ること」に集中している
- いきなり本格的なAI駆動開発をやるより、まずはチャットベースで作る体験から入る方が参加しやすい
- ただし、チャットベースだけで終えると「その先」が見えないため、続編として開発環境編を用意したい

## このファイルで決めること

- 2回シリーズの基本方針
- 第1回と第2回の役割分担
- 第1回の仮タイトル、目的、構成
- 当面の準備項目
- まだ決め切れていない論点

## 企画の基本方針

- 今回の軸は「AIコーディングの進化ステップを体験として見せること」
- 第1回は、チャットでコードを作れる楽しさと、その限界にぶつかるところまでを扱う
- 第2回は、VS Code と AIコーディングツールを使って、ファイル編集・Git・Deploy まで接続する

### 参加者に見せたい流れ

```text
Chatでコードを作る
↓
小さなツールが動く
↓
修正・デバッグがつらくなる
↓
ファイル化したくなる
↓
VS Code / AI coding tools が必要になる
```

この流れがあると、
第1回から第2回への接続に無理がない。

## シリーズ構成案

### 第1回

#### 仮タイトル

- AIでツールを作る入門
- AIツール作成ワークショップ
- Vibe Coding 入門

#### 位置づけ

入門編。
まずはチャットベースで「AIにコードを書かせてツールを作る」体験を作る回。

#### 目的

- AIで小さなツールを作れる実感を持ってもらう
- 参加者自身の仕事や日常に近い題材で、作るハードルを下げる
- チャットベース開発の限界を自然に理解してもらう
- 第2回のAI駆動開発入門につなげる

#### 想定参加者

- 非エンジニア寄りの参加者
- ChatGPT などは触ったことがある人
- AIで何か作ってみたいが、開発環境にはまだ入っていない人

#### 参加後に持ち帰ってほしいもの

- AIで小さなツールを作れるという成功体験
- コード修正やデバッグで何が詰まりやすいかの理解
- なぜ VS Code や AIコーディングツールが必要になるのかの納得感
- 次に自分が作ってみたいテーマの種

#### 扱う題材候補

- Bookmarklet
- HTMLツール
- 簡易Automation
- Excelマクロ

#### 題材例

- YouTubeの速度変更
- ページからテキスト抽出
- URL整形
- Excel整形

#### 紹介できる実例

- 投票ページ
- FastCrop
- 買い物リスト整形GPT

#### 第1回の体験ストーリー

1. AIにコードを書かせる
2. 小さくても動くものが作れる
3. 修正しようとすると急につらくなる
4. チャットだけでは限界があると分かる
5. ファイル化して開発環境に移る意味が見える
6. 第2回でその先をやると理解できる

#### 第1回の構成案

##### 1. 導入

- 今回のシリーズ全体像
- なぜ最初に「AIで作る」体験から始めるのか

##### 2. デモ

- ChatGPT / Claude / Gemini などでコードを作らせる
- 小さなツールがその場で動くところを見せる

##### 3. ワーク

- 参加者が自分の業務や日常で使えそうな小ネタを出す
- その場で作れそうなものを試す

##### 4. 壁の共有

- コード修正がしづらい
- AIが全文を出さなくなる
- どこを直せばいいか分かりづらい
- デバッグがしにくい

##### 5. 解決策の予告デモ

- ファイル化する
- VS Code に持っていく
- Codex / Claude Code / Gemini CLI などで編集する

##### 6. クロージング

- 第2回では何をやるかを見せる
- 参加者に「もっと作りたい」と思わせて終える

#### 仮タイトル

- AI開発の世界を覗く
- AI駆動開発入門

#### 目的

- VS Code を使ったファイルベースの編集に慣れてもらう
- AIコーディングツールの役割を理解してもらう

#### 扱いたい要素

- Codex

```text
↓
Gitで管理する
↓
GitHubや公開環境へつなぐ
```

## 今回の企画で強いポイント

- 単なるAI講座ではなく、「AIで作る」から「AI駆動開発」までの橋渡しになっている
- 非エンジニアにも入れる入口を用意しつつ、次の学びの導線がある
- Kazさんの強みである「非エンジニア目線で、AIツール開発や開発環境を翻訳できること」が活きる

## 現時点の仮結論

- 最初に開催するのは第1回の入門編
- 企画の見せ方は「Vibe Codingそのもの」よりも「AIでツールを作る」が分かりやすい
- 第1回の終盤で VS Code と AIコーディングツールを少しだけ見せ、第2回の予告に使う
- 第2回では VS Code + AI editing + Git + Deploy を扱う
