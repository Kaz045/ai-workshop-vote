# With ワークショップ｜資料構成案

## 0. この資料の位置づけ

この資料は、With ワークショップ当日のスライド構成である。\
主軸は、**AIブームに流されるのではなく、自分のコア × AI で価値を伸ばす**という考え方を土台に、AI時代のものづくり・開発の進め方を参加者に伝えることに置く。

## 2. なぜ今、AIで“作る力”が重要なのか

### 伝えたいこと

- AIブームにただ乗るのではなく、**自分のコアをどう伸ばすか**が大事
- AIそのものが価値なのではなく、AIを使って何を実現するかが価値になる
- これからは「知っている人」よりも、「自分の強みと組み合わせて形にできる人」が強い

### 入れる内容

- AIブームに惑わされない、という視点
- 自分の経験・現場理解・専門性・問題意識とAIを掛け合わせる重要性
- 今日のワークショップは、単なるツール紹介ではなく、その考え方も扱う回であること

---

## 3. いま起きている変化

### 伝えたいこと

- AIによって、非エンジニアでも小さなツールや仕組みを作りやすくなった
- AIがコーディングを解き放ち、誰もが自分に最適な仕組みをつくれる時代が来た
- 一方で、雑に作るだけでは限界にぶつかる

### 入れる内容

- ChatGPTや各種AIツールで簡単なものは作れる時代になった
- 小さな自動化や試作は以前より圧倒的にやりやすい
- ただし、複雑化・継続運用・再利用・改善の段階に進むと難しさが出る

---

## 3.1 AIツール比較（ここに比較表を差し込む）

### 位置づけ

- 「AIに惑わされない」という話をした直後に、現実的な選択肢として各ツールの違いを整理するパート
- ツールに振り回されないために、**特徴と役割を俯瞰して理解する**目的で入れる

### 伝えたいこと

- ツールごとに得意領域・思想が違う
- 正解のツールは1つではなく、目的に応じて使い分ける
- だからこそ「何をやりたいか」が先に必要

### 入れる内容（例）

- ChatGPT / Claude / Gemini / Cursor / Codex などの比較表
- 強み・弱み・用途・思想の違い
- 「どれが最強か」ではなく「どう使い分けるか」

### トークのポイント

- AIに詳しくない人にも全体像を掴ませる
- ここでツールの話をクリアにしておくことで、以降の話（限界・開発環境・設計）が理解しやすくなる

---

## 3.2 ミニワークショップ（共創・ハンズオン）

### 位置づけ

- ツール理解の直後に「実際に触る・考える」ことで、受動から能動へ切り替えるパート
- 参加者の関心・ニーズをその場で引き出し、以降の内容（限界・開発環境）への納得感を高める

### 目的

- 参加者に「自分ごと化」させる
- どんなツールを作っている／作りたいかを可視化する
- 小さく試す体験を通じて、後半の学びにフックを作る

### 進め方（例）

1. 発表者の参考ツールを1〜2例クイック紹介（1〜2分）
2. 参加者に問いかけ
   - 「今どんなツール使ってますか？」
   - 「どんなもの作ってみたいですか？」
3. 数名に共有してもらう（挙手 or チャット）
4. その場で簡単にアイデアを膨らませる／AIに聞いてみる（デモ可）

### トークのポイント

- 正解を出す場ではなく、**試す・話す・広げる場**であることを強調
- 難易度を上げすぎない（1〜5分の軽い体験でOK）
- 「これ、チャットだけでやるとこの後詰まりますよね？」という次のセクションへの布石にする

### 次へのつなぎ（重要）

- 「今みたいに、その場で作れるのはすごく良いんですが…」
- 「これを続けていくと、だんだん詰まってきます」
- 「なぜかというと…」
  → そのまま『チャットだけで作ることの限界』へ接続

---

## 4. チャットだけで作ることの限界

### 伝えたいこと

- AIチャットは強力だが、それだけでは本格開発は苦しくなる
- だからこそ、次のステップが必要になる

### 入れる内容

- その場では動いても、後で直せない
- 仕様が曖昧なまま進みやすい
- ファイル管理・履歴管理・構造化が弱い
- コードや設計が積み上がると、チャットだけでは扱いにくい

---

## 5. 次のステップとしての開発環境

### 伝えたいこと

- AIを本格的に使って何かを作るなら、開発環境の理解が重要になる
- ここで初めて、AI活用が“再現性のあるものづくり”に近づく

### 入れる内容

- VS Code / Cursor / CLI などの位置づけ
- Git / GitHub の役割
- フォルダ構成、ファイル管理、履歴管理の重要性
- AIを使いながら、なぜこうした環境が必要になるのか

---

## 6. 第一部のメッセージ

### 位置づけ

第一部は、**AIでまず作る入口**を扱うパート。

### 伝えたいこと

- まずは小さく作れる
- でも、作れば作るほど次の壁が見える
- その壁を超えるために、開発環境や進め方が必要になる

### 入れる内容

- 小ツール作成・自動化の可能性
- 自分の業務や日常の困りごとから始められること
- 「作って終わり」ではなく、「育てる」視点が必要になること

---

## 7. 第2部：AI時代に価値が高まるもの

### 伝えたいこと

- これから価値が高まるのは、単純なコーディング力だけではない
- 重要なのは、何を作るか、なぜ作るか、どう進めるかを考えられる力

### 入れる内容

- 要件定義
- ヒアリング
- ニーズ把握
- 設計
- 段取り
- 改善の回し方

### 強調ポイント

- AIがコードを補助してくれる時代だからこそ、上流の思考がより重要になる

---

## 8. 第二部のメッセージ

### 位置づけ

第二部は、**本格開発の前に必要な考え方**を扱うパート。

### 伝えたいこと

- いきなり作り始めるのではなく、競技前・設計前・開発前の整理が重要
- 本当に必要なのは、ニーズを聞き、整理し、構造化し、要件に落とす力

### 入れる内容

- ヒアリングの重要性
- 何が課題なのかを見極める視点
- 要件定義の重要性
- 設計段階で考えるべきこと
- AIを使っていても、人間側の整理力が不可欠であること

### 注意

- ここでは特定の題材に決め打ちしない
- 具体テーマに深く入りすぎず、考え方と流れを伝える構成にする

---

## 9. ワークショップ（1〜2部）のまとめ

### 伝えたいこと

- AI時代は、誰でも作り始めやすい
- しかし、価値になるのは「ただ作ること」ではない
- 自分のコアを起点に、必要なものを見抜き、設計し、形にしていく力が重要になる

### 締めのメッセージ案

- AIに振り回されるのではなく、AIを使って自分の強みを伸ばす
- 作れる人になるだけでなく、何をどう作るかを考えられる人になる
- 今日の内容が、その一歩になればうれしい
