# ツール比較表（修正版 / ワークショップ用）

---

## 1. 各ツールの一言ポジション


| ツール     | Wiz向けの表現                |
| ------- | ----------------------- |
| ChatGPT | 幅広い用途に対応しやすい総合型AI       |
| Claude  | 長文読解・整理・設計相談に強い総合型AI    |
| Gemini  | Google連携＋拡張性に強みを持つ総合型AI |


---

## A. ChatGPT / Claude / Gemini


| 観点        | ChatGPT             | Claude          | Gemini                   |
| --------- | ------------------- | --------------- | ------------------------ |
| 一言でいうと    | 汎用性の高いバランス型AI       | 長文・構造理解に強い思考型AI | Google連携＋拡張志向のAI         |
| 向いている使い方  | 日常業務、壁打ち、資料作成、幅広い相談 | 要約、構造化、仕様整理、設計  | Google Workspace連携、データ活用 |
| 曖昧な依頼への対応 | 意図を補完しながら進めやすい      | 整理された依頼で真価を発揮   | 環境連携と組み合わせると強い           |
| 開発との接続    | Codexなど実装系と連携       | Claude Codeと連携  | Gemini CLIやGoogle環境と連携   |
| 参加者向けの見方  | まず1つ選ぶなら有力候補        | 設計・整理を重視する人向け   | Google中心の人に相性が良い         |


---

## B. Codex / Claude Code / Gemini CLI


| 観点       | Codex              | Claude Code       | Gemini CLI          |
| -------- | ------------------ | ----------------- | ------------------- |
| 位置づけ     | 実装特化エージェント（タスク実行型） | 開発全体を回せる統合型エージェント | 拡張前提のAIプラットフォーム型CLI |
| 向いている使い方 | 実装委任、並列作業、自動化      | 設計→実装→修正の一貫開発     | AIを開発環境・パイプラインに組み込む |
| 特徴       | 「作業を任せる」ことに強い      | 読解・設計・実装のバランスが良い  | 巨大コンテキスト＋拡張性が強み     |
| 強み       | 自動実行・効率性           | 長文理解・安定性          | 無料枠・拡張性・プラットフォーム志向  |
| 弱み       | 設計・長文理解はClaudeに劣る  | コスト・やや重い          | CLI完成度・実務精度は発展途上    |
| 参加者向けの見方 | 「AIに作業を任せる」        | 「AIと一緒に開発する」      | 「AIを開発基盤に組み込む」      |


---

### ▼ 一言まとめ（重要）

- Codex → **作業を投げるAI**
- Claude Code → **一緒に開発するAI**
- Gemini CLI → **AIを組み込むための土台**

---

## C. VS Code / Cursor / Antigravity


| 観点       | VS Code     | Cursor        | Antigravity   |
| -------- | ----------- | ------------- | ------------- |
| 位置づけ     | 標準的な開発エディター | AI前提の次世代IDE   | エージェント主導の開発環境 |
| 開発思想     | 人が主導        | 人＋AI協働        | AI主導          |
| 操作主体     | 人間          | 人間（AI補助）      | AI（人は監督）      |
| AIの役割    | 拡張機能として補助   | ペアプロ相棒        | 自律エージェント      |
| 作業スタイル   | 手動開発        | AIと共同開発       | AIに任せて監督      |
| 並列処理     | なし          | 限定的           | 複数エージェントで並列   |
| リスク      | 低           | 中             | 高（暴走リスクあり）    |
| 学習コスト    | 低           | 低〜中           | 中〜高           |
| 向いている人   | 標準から入りたい人   | AI開発を本格活用したい人 | 自動化・高速開発志向の人  |
| 参加者向けの見方 | 無難で入りやすい    | 現時点で最も実用的     | 新しい開発体験として理解  |


---

### ▼ 一言まとめ（ここめちゃ重要）

- VS Code → **普通の開発環境**
- Cursor → **AIと一緒に作る**
- Antigravity → **AIに作らせる**

---

### ▼ 例え（ワークショップ用）

- VS Code → 手運転
- Cursor → ナビ付き運転
- Antigravity → 自動運転＋管制塔

---

- IDE系 → 開発の主導権の違い
