# プロジェクトブリーフ

## 概要

このプロジェクトは、AI ワークショップ企画の需要把握、結果公開、次回開催内容の整理を一つのワークスペースで扱うための基盤です。

## 背景

- 2026年3月に、今後開催する AI ワークショップのテーマ需要を調べるための投票を実施しました
- 投票ページは GitHub Pages 上で公開され、Firebase を使うリアルタイム投票プロトタイプも並行して作られました
- 投票終了後は、結果をもとに次回ワークショップの方向性を具体化する段階に入っています

## 現在のフェーズ

### フェーズ1: 需要調査

- 状態: 完了
- 成果物: `index.html` の最終結果ページ、過去の投票プロトタイプ、関連運用メモ

### フェーズ2: 企画の正式化

- 状態: 進行中
- 成果物: `docs/plans/` の企画メモ、今後整備する `proposals/` と `specs/`

### フェーズ3: 次回開催に向けた具体化

- 状態: 未着手
- 成果物候補: 正式な開催テーマ、進行台本、告知文、必要であれば新しい投票導線

## 主な成果物

- 公開成果物: GitHub Pages 上の結果表示ページ
- 実験資産: Firebase 前提の投票プロトタイプ
- 正史文書: ルート文書と `specs/`
- 履歴資料: `docs/`

## 主な利用者

- 主催者本人
- 次の作業を引き継ぐ AI エージェント
- 将来、企画や実装に参加する協力者

## 成功条件

- 新しい AI エージェントが最小限の読書で現在地を把握できる
- 正史と参考資料が混同されない
- 次回ワークショップの企画方針を `specs/` に昇格させやすい
- 公開ページの扱いと試作版の扱いが明確である

## 非目標

- `docs/chatgpt_exports/` を直接の要件定義として扱うこと
- 旧プロトタイプを仕様更新なしで再公開すること
- すべての履歴資料を直ちに新ディレクトリへ移設すること
