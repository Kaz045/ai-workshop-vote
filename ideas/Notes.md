##　Cursor 3 の紹介
https://cursor.com/blog/cursor-3

ついに来ましたね。Cursor 3 は **2026年4月2日公開**で、Cursor が「IDE」からさらに進んで、**エージェント中心の統合ワークスペース**に寄せてきた、という発表です。 ([Cursor][1])

要点だけいうと、今回の目玉はこのへんです。

* **エージェントを一箇所に集約**
  ローカル/クラウドのエージェントをサイドバーでまとめて扱えます。モバイル、Web、デスクトップ、Slack、GitHub、Linear から起動したものも統合されると案内されています。 ([Cursor][1])

* **複数エージェントを並列で回せる**
  単発の補助ではなく、複数タスクを同時進行させる前提のUIになっています。クラウドエージェントは作業のデモやスクリーンショットも返して、確認しやすくする設計です。 ([Cursor][1])

* **ローカルとクラウドの引き継ぎが速い**
  クラウドで走らせていたセッションをローカルへ持ってきて編集・検証したり、逆にローカルからクラウドへ移して長時間処理を継続したりできます。記事では、そのローカル側の高速反復向けに Composer 2 も挙げています。 ([Cursor][1])

* **commit から PR までをそのまま流せる**
  新しい diff view で変更レビューし、stage、commit、PR 管理までつなげる流れを押し出しています。 ([Cursor][1])

* **IDEの良さも残している**
  ファイル閲覧、定義ジャンプ、統合ブラウザ、Marketplace のプラグインなど、従来の IDE 的な深掘りもしやすいとしています。 ([Cursor][1])

一言でまとめると、**「コードを自分で細かく触る場」から、「複数エージェントを指揮して成果物をレビューする場」へ、Cursor が本格的に舵を切った**感じです。記事でも、ソフトウェア開発が「エージェントが大半のコードを書く方向」に進んでいるという前提で、Cursor 3 をそのための統合環境として位置づけています。 ([Cursor][1])

試すには、記事上では **Cursor をアップグレードして `Cmd+Shift+P -> Agents Window`** を開くよう案内されています。 ([Cursor][1])
